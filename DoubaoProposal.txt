# Windows+OpenCode+本地Whisper+无焦点浮窗语音输入 完整执行计划
## 核心目标
在OpenCode Desktop中实现「无焦点浮窗按钮→按住说话→本地Whisper识别→文字直接插入OpenCode光标」，全程开源、本地处理、不抢焦点、无需粘贴。

## 一、前置准备
### 1. 环境要求
- Windows 10/11（64位）
- Python 3.8+（推荐3.10，官网：https://www.python.org/downloads/windows/）
- 麦克风（内置/外置均可）
- 至少2GB空闲磁盘（Whisper模型大小：base≈1GB，small≈1.5GB，medium≈4GB）
- 管理员权限（用于运行无焦点浮窗）

### 2. 关键开源项目（已验证网址有效性）
| 项目名称       | 用途                     | 开源地址（可直接访问）                          |
|----------------|--------------------------|-------------------------------------------------|
| Whisper        | 本地语音识别核心         | https://github.com/openai/whisper               |
| WhisperBoard   | 无焦点悬浮窗（适配Whisper）| https://github.com/AndreHayhurst/WhisperBoard   |
| pywin32        | Windows无焦点窗口实现    | https://github.com/mhammond/pywin32             |
| Flask          | 本地语音识别API服务      | https://github.com/pallets/flask                |

## 二、分步执行计划（共6步，全程可落地）


### 步骤2：部署本地Whisper语音识别API服务
#### 2.1 创建本地API服务文件
在桌面新建文件夹`VoiceInput`，里面新建`local_stt_server.py`，内容如下：
```python
# local_stt_server.py
from flask import Flask, request, jsonify
import whisper
import tempfile
import os
import threading

app = Flask(__name__)
# 加载本地Whisper模型（small适合多数电脑，低配换base，高配换medium）
model = whisper.load_model("small", device="cpu")  # 有NVIDIA GPU可改device="cuda"
model_lock = threading.Lock()  # 防止多线程冲突

@app.route("/transcribe", methods=["POST"])
def transcribe():
    try:
        # 接收前端上传的音频文件
        if "audio" not in request.files:
            return jsonify({"error": "无音频文件"}), 400
        
        audio_file = request.files["audio"]
        # 临时保存音频（本地处理，不持久化）
        with tempfile.NamedTemporaryFile(suffix=".wav", delete=False) as f:
            audio_path = f.name
            audio_file.save(audio_path)
        
        # 本地识别（加锁防止并发问题）
        with model_lock:
            result = model.transcribe(
                audio_path,
                language="zh",  # 强制识别中文
                fp16=False,     # CPU模式设为False，GPU设为True
                verbose=False
            )
        
        # 删除临时文件
        os.unlink(audio_path)
        return jsonify({"text": result["text"].strip()})
    
    except Exception as e:
        return jsonify({"error": str(e)}), 500

if __name__ == "__main__":
    # 启动本地API服务（仅本机可访问，安全）
    app.run(host="127.0.0.1", port=9999, debug=False, threaded=True)
```

#### 2.2 启动API服务
- 打开CMD，进入`VoiceInput`文件夹：
```bash
cd Desktop\VoiceInput
python local_stt_server.py
```
- 看到`Running on http://127.0.0.1:9999`即为启动成功（保持CMD窗口打开，关闭则服务停止）。

### 步骤3：部署无焦点浮窗（核心：不抢OpenCode焦点）
#### 3.1 下载适配的开源浮窗工具（WhisperBoard）
- 访问验证后的地址：https://github.com/AndreHayhurst/WhisperBoard
- 点击「Code」→「Download ZIP」下载源码，解压到`VoiceInput`文件夹下。

#### 3.2 配置WhisperBoard（适配本地API）
- 打开WhisperBoard源码中的`app.js`（或`main.js`），找到音频上传地址，修改为：
```javascript
// 原地址可能是远程API，替换为本地服务
const API_URL = "http://127.0.0.1:9999/transcribe";
```
- 找到窗口配置代码，确保添加Windows无焦点属性：
```javascript
// 关键：让浮窗不抢焦点（WhisperBoard已内置，只需确认）
win = new BrowserWindow({
  width: 80,
  height: 80,
  frame: false,
  alwaysOnTop: true,
  skipTaskbar: true,
  webPreferences: { nodeIntegration: true },
  // Windows无焦点核心配置
  win.setAlwaysOnTop(true, "screen-saver");
  win.setIgnoreMouseEvents(false, { forward: true });
  win.setFocusable(false); // 关键：不可获取焦点
});
```

#### 3.3 打包/运行WhisperBoard
- 若不想打包，直接运行（需安装Node.js，https://nodejs.org/zh-cn/download/）：
```bash
# 进入WhisperBoard目录
cd Desktop\VoiceInput\WhisperBoard
# 安装依赖
npm install
# 启动浮窗
npm start
```
- 启动后会出现一个悬浮麦克风图标，**点击时不会抢走OpenCode的焦点**（核心效果）。

### 步骤4：配置OpenCode（无需修改源码）

- 无需修改OpenCode任何文件，浮窗会直接将文字输入到当前焦点位置。

## 三、常见问题解决
1. **浮窗点击后OpenCode焦点丢失**：
   - 确认WhisperBoard的窗口配置中`setFocusable(false)`已开启；
   - 以管理员身份运行WhisperBoard。

2. **语音识别慢**：
   - 将Whisper模型从small换成base（速度更快，准确率略降）；
   - 有NVIDIA GPU的话，安装CUDA（https://developer.nvidia.com/cuda-downloads），修改`local_stt_server.py`中`device="cuda"`。

3. **识别结果乱码/无结果**：
   - 确认麦克风权限已开启（Windows设置→隐私和安全性→麦克风）；
   - 确认ffmpeg已安装并添加到环境变量。

## 总结
1. **核心架构**：本地Whisper API服务（Python）+ 无焦点浮窗（WhisperBoard）+ OpenCode（无需修改），全程本地、开源、不抢焦点。
2. **关键步骤**：启动本地API服务→运行无焦点浮窗→OpenCode光标定位→按住浮窗说话→文字直接输入。
3. **核心保障**：浮窗通过`setFocusable(false)`实现不抢焦点，Whisper通过本地API实现离线识别，最终文字直接注入OpenCode光标位置。
