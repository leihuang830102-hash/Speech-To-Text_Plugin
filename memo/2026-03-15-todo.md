# 待办事项 - 2026-03-15

## 1. Python 启动优化

### 问题
当前 Python 启动需要 ~4 秒，导致用户体验差：
- 用户按住后要等待很久才能开始说话
- 释放太早会导致录音不完整

### 方案选项

#### 方案 A: Python 常驻进程（推荐）
- App 启动时预加载 Python 进程
- Python 保持运行，等待录音指令
- 通过 stdin/stdin 或 socket 通信
- **优点**：启动一次，后续秒开
- **缺点**：占用内存，需要进程管理

#### 方案 B: 使用更轻量的 STT 引擎
- 考虑 moonshine-onnx（已经支持）
- 或使用 whisper.cpp（C++ 原生，启动快）
- **优点**：启动快
- **缺点**：可能需要额外配置

#### 方案 C: WebAssembly STT
- 使用 whisper.wasm 在浏览器端运行
- 不需要 Python
- **优点**：无需外部进程
- **缺点**：首次加载模型慢

### 决策
**优先尝试方案 A** - Python 常驻进程

├── src/
│   ├── index.tsx          # 主入口（OpenCode 插件）
